HR ERP Fix Notes
================
Fixed database errors shown in screenshots:

1. PostgreSQL syntax error near current_role
   - Renamed employee role database column from current_role to role_title.
   - current_role conflicts with PostgreSQL CURRENT_ROLE keyword/function usage.

2. relation "employees" does not exist
   - Updated database/init.sql to create employees table correctly.
   - Added application-level schema bootstrap in app/main.py so tables are created even if an old Docker volume exists or init.sql did not run.

How to run:
1. Extract this fixed ZIP.
2. Run Launch_HR_ERP.bat.
3. If Docker is already running the old version, run Stop_HR_ERP.bat first, then Launch_HR_ERP.bat.
4. If you still see old database errors, run this once from the project folder to reset the broken old database volume:
   docker-compose down -v
   docker-compose up -d --build

Note: docker-compose down -v deletes the existing local database volume, so use it only if you do not need old test data.
